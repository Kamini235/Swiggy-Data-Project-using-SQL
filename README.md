# Swiggy-Data-Project-using-SQL
Swiggy Data Analysis Project
This project uses data from swiggy food delivery platform to analyse key metrics such as Order volume, Rating, Location, Restaurant rating. This analysis performed using SQL  
Swiggy data Analytics: A dynamic, interactive data visualization tool built to explore swiggy data worldwide—focusing on regional comparisons and other important Insight.
Purpose
This project is focused on analysis data related to food delivery services provided by swiggy. The project Aim to Provided insight into trend & pattern in food delivery order, category wise, location wise and restaurant wise performance.

Key questions such as: Which locations Provides the most Revenue? Which is more ordered dish by customers? Which Restaurant have the most Order placed? … are difficult to answer quickly with raw data.
• Goal of the Dashboard To deliver an interactive visual tool that: Enables users to explore swiggy data. Supports decisions such as future planning, regional marketing, or infrastructure investment. Uncovers trends in terrain, accessibility, and customers like by region and restaurant.

Data source
The data used in this project is obtained from swiggy website requested paged by performing web scrapping. Which include data in table having column order, location, dish name, restaurant, rating .. etc.

Usages
To use this project, you need to have SQL server installed in your machine along with sql server management studio and Visualisation tool Power-BI. You can simply download or clone the project repository from GitHub & run the file to perform analysis.

Insights
Created and optimized MySQL databases to efficiently store and retrieve data for service providers and user profiles.
Analysed Restaurant, Dish name, Category and sales data to identify pricing inefficiencies and customer dependency. And Cleaned data using SQL
